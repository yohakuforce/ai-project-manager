# パイロット運用 内部チェックリスト（社内限定）

> このファイルは**社内運用判断**をまとめた内部メモです。外部公開リポにする際は
> このファイル（`docs/internal/`）を除外してください。技術的な導入手順は
> [`../deploy-windows.md`](../deploy-windows.md) を参照。

## 運用開始前の社内判断・承認事項

- **情報システム / 法務のパイロット承認** — 顧客データを扱うため、取込先（社内 PC）の
  データ境界とアクセス権限について承認を得る。
- **公開判断（依存パッケージ側）**:
  - Context-Hub の REST 応答 camelCase 化は v0.3.0 の破壊的変更。PyPI 公開状況に応じて
    AI-PM 側の追従バージョンを確定する。
  - `@yohakuforce/core` の npm 再公開可否。

## データ境界（再掲）

顧客機密は Context-Hub の取込先（社内 PC）に閉じる。AI-PM が REST 越しに受け取るのは
抽象化済みの構造化データのみ。生データを外部 LLM / SaaS に転記しない方針は両層で維持する
（LLM はサブスク CLI もしくはローカル LLM で運用＝外部 API へ生データを出さない）。
